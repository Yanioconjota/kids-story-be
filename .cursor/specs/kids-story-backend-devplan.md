# Development Plan: Kids Story Backend MVP (Bootstrap)

> Source spec: `.cursor/specs/kids-story-backend-cursor-prompt.md`
> Generated via the `spec-to-devplan` skill, aligned with `.cursor/rules/`.

## Overview

Bootstrap a Dockerized FastAPI microservices backend for the **Kids Story** platform. Four
services orchestrate a single **Story Generation** flow: a **Story Request** is validated by
moderation, generated by the LLM service (mocked), and persisted as a **Story** in MongoDB.
Redis is wired into compose for future caching but not yet used. No auth, frontend, image,
narration, or queues (per MVP constraints in `000-project-context.md`).

## Approved Decisions

- **`shared/` at repo root**, imported via Docker `build.context: .` (single source of truth for contracts).
- **Thin shared package**: start with `contracts/` + `errors/`; defer `logging/`/`config/` until a service needs more than basic env reads.
- **Repo-root layout** (no extra `kids-story-backend/` nesting).

## Shared Contracts (domain language)

Defined once in `shared/contracts/story.py`, imported by every service to keep the REST
contract in sync:

```python
class StoryRequest(BaseModel):
    child_theme: str = Field(..., max_length=120)        # Child Theme
    character_name: str = Field(..., max_length=80)      # Character Name
    prompt: Optional[str] = Field(default=None, max_length=500)  # Story Prompt

class ModerationResult(BaseModel):
    approved: bool
    reason: Optional[str] = None

class Story(BaseModel):
    id: str = Field(default_factory=lambda: str(uuid4()))
    child_theme: str
    character_name: str
    prompt: Optional[str] = None
    content: str
    created_at: datetime = Field(default_factory=datetime.utcnow)
```

> Domain-language note: uses Story / Story Request / Child Theme / Character Name /
> Moderation Result — no chat/conversation/message terms.

## Story Creation Flow

```txt
Client → POST /stories (api-gateway)
  → POST /moderate (moderation-service)      # mock: { approved: true }
  → POST /generate  (llm-service)            # mock: { content: "Once upon a time..." }
  → POST /stories   (story-service) → MongoDB # persist Story, return saved doc
  → 201 with Story  (back to client)
```

- If moderation returns `approved: false`, gateway raises `HTTPException(400, detail=...)` and stops the flow.
- All inter-service calls use `httpx.AsyncClient(timeout=...)`; service URLs come from env vars.

## Implementation Checklist

### Shared
- [ ] `shared/contracts/story.py` — `StoryRequest`, `Story`, `ModerationResult` (typed, `Field()` defaults, UUID `id`).
- [ ] `shared/errors/exceptions.py` — small helpers for consistent error `detail` shapes.

### Data Models (Pydantic)
- [ ] All request/response bodies typed via shared contracts — no `dict`/`Any`.
- [ ] `Story.id` is UUID string; `_id` left to MongoDB.

### API Endpoints (FastAPI)
- [ ] Every service: `GET /health` → `{"status": "ok", "service": "<name>"}`.
- [ ] `api-gateway`: `async def` `POST /stories` orchestrating the flow; specific `HTTPException` codes (400 on moderation reject, 502 on downstream failure).
- [ ] `moderation-service`: `POST /moderate` → mock `ModerationResult(approved=True)`.
- [ ] `llm-service`: `POST /generate` → mock `Story` content from the request.
- [ ] `story-service`: `POST /stories` (persist) + `GET /stories/{id}` (retrieve, 404 if missing).
- [ ] Endpoints kept in `main.py` (each service well under 10 endpoints).

### Inter-service Communication
- [ ] `api-gateway/app/clients.py` uses `httpx.AsyncClient` with timeouts; no blocking `requests`.
- [ ] Service base URLs from env: `MODERATION_URL`, `LLM_URL`, `STORY_URL`.
- [ ] Gateway orchestrates; each service owns its responsibility (boundaries respected).

### Persistence (MongoDB) — story-service
- [ ] `database.py`: connect using `MONGO_URI`; expose `stories_collection`.
- [ ] `ensure_indexes()` on startup: unique index on `id`, index on `created_at` (DESC).
- [ ] Insert via `insert_one`; retrieve via `find_one({"id": id})`.

### Caching (Redis)
- [ ] Wired in compose + `REDIS_URL` env only; no cache logic this iteration (MVP). Documented as a future hook.

### Streaming (SSE)
- [ ] Out of scope this iteration (llm-service mock is synchronous). The `llm-service` is the future home for SSE per `003-sse-streaming.md`.

### Error Handling
- [ ] Pydantic validates request bodies at each boundary.
- [ ] Specific, actionable `HTTPException` messages (no generic 500s).

### Testing (pytest)
- [ ] `services/story-service/tests/test_stories.py` — create + retrieve + 404 via `TestClient`.
- [ ] `services/api-gateway/tests/test_flow.py` — happy path (mock downstreams) + moderation-reject path.
- [ ] `moderation-service` / `llm-service` — `/health` + mock endpoint smoke tests.

### Docker / Config
- [ ] Per-service `Dockerfile` (python slim base, install `requirements.txt`, `COPY shared/`, uvicorn entrypoint).
- [ ] Per-service `.env.template` (URLs, `MONGO_URI`, `REDIS_URL`, `PORT`) — never commit `.env`.
- [ ] `infra/docker-compose.yml`: 4 services + `mongo` + `redis`, with `build.context: .`, ports, `env_file`, `depends_on`.
- [ ] `.gitignore` covers `.env`, `__pycache__`, `*.pyc`.
- [ ] `README.md`: run instructions (`docker compose -f infra/docker-compose.yml up --build`) + the flow diagram.
