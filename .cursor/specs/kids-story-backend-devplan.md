# Development Plan: Kids Story Backend MVP (Bootstrap)

> Source spec: `.cursor/specs/kids-story-backend-cursor-prompt.md`
> Generated via the `spec-to-devplan` skill, aligned with `.cursor/rules/`.

## Overview

Bootstrap a Dockerized FastAPI microservices backend for the **Kids Story** platform. Four
services orchestrate a single **Story Generation** flow: a **Story Request** is validated by
moderation, generated by the LLM service (mocked), and persisted as a **Story** in MongoDB.
Redis is wired into compose for future caching but not yet used. No auth, frontend, image,
narration, or queues (per MVP constraints in `000-project-context.md`).

## Approved Decisions

- **`shared/` at repo root**, imported via Docker `build.context: .` (single source of truth for contracts).
- **Thin shared package**: start with `contracts/` + `errors/`; defer `logging/`/`config/` until a service needs more than basic env reads.
- **Repo-root layout** (no extra `kids-story-backend/` nesting).

## Shared Contracts (domain language)

Defined once in `shared/contracts/story.py`, imported by every service to keep the REST
contract in sync:

```python
class StoryRequest(BaseModel):
    child_theme: str = Field(..., max_length=120)        # Child Theme
    character_name: str = Field(..., max_length=80)      # Character Name
    prompt: Optional[str] = Field(default=None, max_length=500)  # Story Prompt

class ModerationResult(BaseModel):
    approved: bool
    reason: Optional[str] = None

class Story(BaseModel):
    id: str = Field(default_factory=lambda: str(uuid4()))
    child_theme: str
    character_name: str
    prompt: Optional[str] = None
    content: str
    created_at: datetime = Field(default_factory=datetime.utcnow)
```

> Domain-language note: uses Story / Story Request / Child Theme / Character Name /
> Moderation Result вЂ” no chat/conversation/message terms.

## Story Creation Flow

```txt
Client в†’ POST /stories (api-gateway)
  в†’ POST /moderate (moderation-service)      # mock: { approved: true }
  в†’ POST /generate  (llm-service)            # mock: { content: "Once upon a time..." }
  в†’ POST /stories   (story-service) в†’ MongoDB # persist Story, return saved doc
  в†’ 201 with Story  (back to client)
```

- If moderation returns `approved: false`, gateway raises `HTTPException(400, detail=...)` and stops the flow.
- All inter-service calls use `httpx.AsyncClient(timeout=...)`; service URLs come from env vars.

## Implementation Checklist

### Shared
- [ ] `shared/contracts/story.py` вЂ” `StoryRequest`, `Story`, `ModerationResult` (typed, `Field()` defaults, UUID `id`).
- [ ] `shared/errors/exceptions.py` вЂ” small helpers for consistent error `detail` shapes.

### Data Models (Pydantic)
- [ ] All request/response bodies typed via shared contracts вЂ” no `dict`/`Any`.
- [ ] `Story.id` is UUID string; `_id` left to MongoDB.

### API Endpoints (FastAPI)
- [ ] Every service: `GET /health` в†’ `{"status": "ok", "service": "<name>"}`.
- [ ] `api-gateway`: `async def` `POST /stories` orchestrating the flow; specific `HTTPException` codes (400 on moderation reject, 502 on downstream failure).
- [ ] `moderation-service`: `POST /moderate` в†’ mock `ModerationResult(approved=True)`.
- [ ] `llm-service`: `POST /generate` в†’ mock `Story` content from the request.
- [ ] `story-service`: `POST /stories` (persist) + `GET /stories/{id}` (retrieve, 404 if missing).
- [ ] Endpoints kept in `main.py` (each service well under 10 endpoints).

### Inter-service Communication
- [ ] `api-gateway/app/clients.py` uses `httpx.AsyncClient` with timeouts; no blocking `requests`.
- [ ] Service base URLs from env: `MODERATION_URL`, `LLM_URL`, `STORY_URL`.
- [ ] Gateway orchestrates; each service owns its responsibility (boundaries respected).

### Persistence (MongoDB) вЂ” story-service
- [ ] `database.py`: connect using `MONGO_URI`; expose `stories_collection`.
- [ ] `ensure_indexes()` on startup: unique index on `id`, index on `created_at` (DESC).
- [ ] Insert via `insert_one`; retrieve via `find_one({"id": id})`.

### Caching (Redis)
- [ ] Wired in compose + `REDIS_URL` env only; no cache logic this iteration (MVP). Documented as a future hook.

### Streaming (SSE)
- [ ] Out of scope this iteration (llm-service mock is synchronous). The `llm-service` is the future home for SSE per `003-sse-streaming.md`.

### Error Handling
- [ ] Pydantic validates request bodies at each boundary.
- [ ] Specific, actionable `HTTPException` messages (no generic 500s).

### Testing (pytest)
- [ ] `services/story-service/tests/test_stories.py` вЂ” create + retrieve + 404 via `TestClient`.
- [ ] `services/api-gateway/tests/test_flow.py` вЂ” happy path (mock downstreams) + moderation-reject path.
- [ ] `moderation-service` / `llm-service` вЂ” `/health` + mock endpoint smoke tests.

### Docker / Config
- [ ] Per-service `Dockerfile` (python slim base, install `requirements.txt`, `COPY shared/`, uvicorn entrypoint).
- [ ] Per-service `.env.template` (URLs, `MONGO_URI`, `REDIS_URL`, `PORT`) вЂ” never commit `.env`.
- [ ] `docker-compose.yml` (repo root): 4 services + `mongo` + `redis`, with `build.context: .`, ports, `env_file`, `depends_on`.
- [ ] `.gitignore` covers `.env`, `__pycache__`, `*.pyc`.
- [ ] `README.md`: run instructions (`docker compose up --build`) + the flow diagram.

## Post-MVP Roadmap

Detailed per-feature design notes live in the knowledge base: `.cursor/docs/`.

### Recommended order: LLM в†’ SSE streaming в†’ Caching

| # | Feature | Service(s) | Why now | KB doc |
|---|---------|-----------|---------|--------|
| 1 | **Real LLM integration** | `llm-service` | Core product value; replaces the string-template mock | `.cursor/docs/01-llm-integration.md` |
| 2 | **SSE streaming** | `llm-service`, `api-gateway` | Stream tokens as generated instead of waiting for the full story | `.cursor/docs/02-sse-streaming.md` |
| 3 | **Real moderation** | `moderation-service` | Replace `approved: True` mock with real child-safety checks | `.cursor/docs/03-real-moderation.md` |
| 4 | **Redis caching** | `api-gateway` / `llm-service` | Redis already wired; cache identical Story Requests to save tokens | `.cursor/docs/04-redis-caching.md` |
| 5 | **Story listing + pagination** | `story-service` | `GET /stories`; `created_at` index already in place | `.cursor/docs/05-story-listing.md` |

### Later (explicitly out of current MVP scope)

| # | Feature | Notes |
|---|---------|-------|
| 6 | Lightweight user identity | Optional client-generated `user_id` on `StoryRequest` (no login) to filter stories per device |
| 7 | Full authentication | JWT / sessions вЂ” productization step; currently excluded by `000-project-context.md` |

### Cross-cutting principles for all roadmap items

- Keep the four-service boundaries intact; the gateway orchestrates, services own their data.
- New request/response shapes go through shared Pydantic contracts in `shared/contracts/`.
- Inter-service calls stay on `httpx.AsyncClient` with timeouts.
- Cache and moderation failures must degrade gracefully вЂ” never break the core flow.
- Each feature ships with pytest coverage (behavior, not internals).
